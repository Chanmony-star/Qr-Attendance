## Communication
- **Issues**: Bugs and features
- **Discussions**: Questions and ideas
- **PRs**: Code reviews

## Rules
1. Never push directly to `main`
2. Never push directly to `develop`
3. Always work on feature branches
4. Pull before starting work
5. Commit small changes
6. Use clear commit messages
7. Create PR when done
8. Get review before merging
9. Ask for help if stuck > 1 hour
10. Test before pushing
